# FlowTask V58.12.6

Release focus: clean workspace catalog logic, client deletion, organization deletion/reactivation alignment, and task/project form persistence.

## Main fixes
- Added migration `0038_v58_12_6_database_sanitization_foundation.sql`.
- Added organization columns expected by the app: `purge_after` and `reactivated_at`.
- Fixed `client_deleted` activity log enrichment so it does not reattach `client_id` from metadata.
- Added `delete_workspace_client(p_client_id uuid)` RPC.
- Catalog loading now includes global defaults plus scoped workspace rows, deduped by normalized name.
- Task/project department resolver accepts code or name and falls back to global catalog rows.
- Client delete uses the new RPC with fallback to direct delete.
- Hard-delete organization cleanup removes org tasks/projects/activity logs before deleting the organization.
- RLS policies allow organization workspace access for projects/tasks beyond only `owner_id`.

## Deployment order
1. Backup/snapshot Supabase.
2. Apply migration `0038_v58_12_6_database_sanitization_foundation.sql`.
3. Deploy app.
4. Smoke test: task/project edit country+department, delete client, soft-delete/reactivate organization.
