# V58.0 — Organization bootstrap, admin inicial y consistencia operativa

## Cambios clave
- Alta inicial real de organización desde la app.
- El usuario creador queda como `admin_global` y `is_default = true`.
- Se crea suscripción inicial del workspace.
- Roles con copy más claro (`Owner · Admin`, `Manager`, `Member`, `Viewer`).
- Proyectos y tareas nuevas exigen organización activa para evitar inconsistencias.
- Clientes permiten guardado en modo personal usando `account_owner_id`.
- Onboarding y settings muestran mejor el contexto cuando no existe organización activa.
